5. Explanation of how Linux manages file I/O
   Linux uses file descriptors to manage file input and output.  When a process starts it gets three file descriptors. The Linux file descriptor 0 
   is for input the descriptor 1 is for output and the descriptor 2 is for error messages.When a process wants to access a file it uses commands 
   like open, read, write and close. The Linux kernel handles these commands and Makes it easy for files, devices, pipes and sockets to work 
   together.Linux also lets you redirect input and output. This means you can send output and error messages to files of the terminal. You can use 
   symbols like > and 2> to do this. This is really useful for keeping track of what is happening with an application and, for fixing problems.
